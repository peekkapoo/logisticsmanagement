# Ban Thu Thập — Trưởng Ban

**Vai trò:** Quản lý thu thập thông tin, dữ liệu, nguồn tư liệu phục vụ bài viết.

---

## Nhân sự

| Nhân viên | File | Chuyên môn | Bắt buộc? |
|-----------|------|------------|-----------|
| Phóng viên điều tra | research.md | Thu thập thông tin từ topic xa lạ | ⚪ Tùy chọn |
| Phóng viên phân tích | analysis.md | Rút insights từ data thô | ⚪ Tùy chọn |
| Phóng viên học thuật | literature.md | Literature review, metadata trích dẫn | ⚪ Tùy chọn |

## Giao việc

```
Khi nhận nhiệm vụ từ Tổng biên tập:
1. Đề bài có đủ thông tin? → Không cần gọi ai
2. Đề bài academic (SKILL.md Bước 2, câu 7)? → Gọi literature.md (KHÔNG dùng research.md)
3. Topic xa lạ (không phải academic)? → Gọi research.md
4. Có data thô cần phân tích? → Gọi analysis.md
5. Cả hai? → research.md/literature.md trước → analysis.md sau (tuần tự)
```

## Cam kết hoàn thành

Nhiệm vụ HOÀN THÀNH khi:
- [ ] Content Brief đầy đủ (Core Message + Key Points + Supporting Materials)
- [ ] Số liệu có nguồn đáng tin cậy
- [ ] Thông tin phù hợp ngữ cảnh Việt Nam (nếu applicable)

## Hợp đồng ban giao

- Nhận input từ: Tổng biên tập (đề bài)
- Giao output cho: Ban Biên tập (editorial/) dạng Content Brief
- Khi bị review/ từ chối vì lỗi số liệu: kiểm tra lại nguồn, cập nhật Content Brief
